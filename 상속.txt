< 상속 >

메서드나 생성자에서 중복되는 데이터가 있을 때는 유지보수를 용이하게 하기위해
어떻게 관리하나? ---> 변수선언

클래스에서 메서드나 생성자에 중복되는 코드가 있을 때는 유지보수를 용이하게
하기 위해서 어떻게 관리하나? --> 메서드

프로젝트에서 메서드나 변수 또는 상수가 중복될 때, 유지보수를 용이하게 하기 위해
어떻게 관리해야 할까? --> 상속 (부모클래스를 지정)

상속은 부모클래스의 자원(변수,상수, 메서드 -생성자는 포함x)을 자식클래스에서 임의로
사용할 수 있게 하는 기술이다. 상속을 함으로써 한 프로젝트 내에서 클래스간에 공통으로
갖고있는 자원에 대해 유지나 보수가 용이하게 한다.

상속을 적용하더라도 같은 개념(상위 개념)이 같아야한다.
 ex) 자동차는 자동차끼리, 가전은 가전끼리

상속 표현에 사용되는 키워드는? 클래스명 옆에 extends 부모클래스 이름
			ex) public class Sonata extends Car

부모클래스로는 외부에 객체를 만들지 않는다. => new 부모클래스() -> x
: 유지 보수를 위해서 만들었기 때문에 (단, 자식 클래스에서는 super();의 형태로 형성이 가능)

부모클래스에는 디폴트생성자까지만 있으면 됨

상속의 표현은 부모클래스와 자식클래스 중 어디에 한다? -> 자식 클래스에 한다!

자바의 최상위 부모클래스 ? Object

자바는 단일상속을 원칙으로 한다 : 상속받을 수 있는 클래스는 한개뿐!
( * 예외 : interface 는 다중상속을 지원한다. )

부모클래스(super클래스/ 기본클래스) : 자식클래스(sub클래스/ 유도클래스)

super( ) : 부모클래스의 생성자호출 =>부모 객체 (클래스 자체로는 활용할 수 없기 때문에)

부모 클래스의 변수명과 자식클래스의 변수명이 같을 경우,
누가 우선적으로 인식되는가? --> 자식클래스의 변수가 우선적으로 인식된다.
부모클래스의 변수명(또는 자원)을 우선으로 인식하고 싶을 때 -> super.변수명(또는 자원명)

 : 객체를 생성하는 다양한 방법. 
   하나의 타입에 다양한 객체를 대입하여 다양한 기능을 이용할 수 있도록 함

       Car       sonata2     =   new Sonata();
부모클래스 변수명(객체명) = new 자식생성자();

다형성으로 생성된 객체의 자료형은 무엇일까?
 => 다형성으로 만들어진 객체의 자료형은 부모클래스의 자료형으로 통일된다.


<상속에서의 메서드 오버라이딩>

상속 관계가 성립하는 클래스에서 부모 클래스가 갖고있는 메서드의 내용을
자식클래스에서 재정의(수정/삭제/추가)하는 것

다형성으로 생성된 객체의 경우, 자료형이 부모자료형이어도 그 객체가 실행하는 
메서드는 오버라이딩된 메서드다. 

부모클래스에서 만들어진 메서드는 자식클래스에서 사용할 수 있지만
자식클래스에서 만들어진 메서드는 자식클래스에서 사용할 수 없고
자식클래스에서 수정된 메서드는 메서드 오버라이딩을 해줘야만 수정된 내용이 적용된다
메서드 오버라이딩이 된 후에는 부모클래스에서 사용해도 자식클래스에서 수정된 내용으로 실행된다.

- 메서드 오버로딩과 오버라이딩
 : 오버로딩은 - 같은 클래스 내에서 이름이 같은 메서드가 파라미터 자료형에 따라 인정되는 문법
   오버라이딩 - 상속관계가 성립되는 부모클래스와 자식 클래스에서 자식클래스가 부모클래스의
	      메서드를 재정의(수정.삭제.추가)하는 것을 의미함
	      다형성으로 생성된 객체의 경우, 자료형이 부모클래스의 자료형이라도 
	      그 객체가 호출한 메서드는 오버라이딩이 된 메서드가 호출된다.



다형성을 이용해서 객체를 만들었을 때 단점

 : 다형성으로 만들어진 객체는 자식클래스에서 만들어진 고유 자원에는 접근할 수 없음
 왜냐하면 자료형이 부모클래스의 자료형이기 때문에

 이를 해결할 수 있는 방법은 instanceof 라는 키워드나 getClass()메서드 둘중 하나를 사용
